## Highlights

- Move terminals between slots and workspaces without losing the running session.
- Drop a terminal onto the Agent panel to attach its live context.
- BLXCode now restores empty workspaces more reliably on Linux.

## New

- **Terminal drag and drop**: Reorder terminal slots, move a running terminal into another workspace, or create a new workspace from an existing terminal.
- **Terminal to Agent context**: Drag a terminal onto the Agent tab so the Agent can understand what is happening in that session.

## Improved

- Right-panel content now mounts more lazily, making startup calmer and reducing hidden work.
- Terminal drag previews are clearer, with dashed drop targets and better visual feedback.

## Fixed

- Fixed a startup path where BLXCode could remain stuck after restoring an empty workspace on Linux.
- Removed stale terminal drag-and-drop debug code and related warnings.

## Good to know

- Existing terminal sessions are preserved during drag-and-drop moves.
- The technical changelog is still available in `CHANGELOG.md` if you want the full implementation details.
