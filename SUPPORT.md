# Support

Need help with BLXCode? Start here before opening a bug report.

## Self-service

1. **[Documentation](docs/README.md)** — user and developer guides in the repository
2. **[GitHub Wiki](https://github.com/Bitslix/BLXCode/wiki)** — same docs, browsable on GitHub
3. **[Troubleshooting](docs/user/troubleshooting.md)** — common build, startup, keyring, terminal, and browser issues
4. **[Getting started](docs/user/getting-started.md)** — install, first workspace, settings, where data lives
5. **[Releases](https://github.com/Bitslix/BLXCode/releases)** — download installers and read release notes
6. **[CHANGELOG](CHANGELOG.md)** — what changed in each version

## Community channels

| Need | Where |
|------|-------|
| Usage or setup question | [Discussions → Q&A](https://github.com/Bitslix/BLXCode/discussions/categories/q-a) |
| Feature idea or feedback | [Discussions → Ideas](https://github.com/Bitslix/BLXCode/discussions/categories/ideas) |
| Confirmed bug or regression | [Issues → Bug report](https://github.com/Bitslix/BLXCode/issues/new?template=bug_report.yml) |
| Feature request (tracked) | [Issues → Feature request](https://github.com/Bitslix/BLXCode/issues/new?template=feature_request.yml) |

Search [existing discussions](https://github.com/Bitslix/BLXCode/discussions) and [issues](https://github.com/Bitslix/BLXCode/issues) first — your question may already be answered.

## Reporting bugs effectively

Include:

- **BLXCode version** (Settings → App, or release tag)
- **Platform** (Linux / macOS / Windows + version)
- **Steps to reproduce** and expected vs actual behavior
- **Logs** from the terminal or browser devtools when relevant

## Contributing code

See [CONTRIBUTING.md](CONTRIBUTING.md) and [docs/developer/contributing.md](docs/developer/contributing.md).

## Security

BLXCode is a local desktop app. Do not paste API keys, `.env` files, or private workspace paths into public issues or discussions.
